# 登录页与图标使用指南

> 适用范围：`.NET + Avalonia` 跨平台项目（Desktop + Android）
>
> 目标：用一套资源源头，稳定生成 **Windows 系统图标**、**Android 应用图标**、**Android 开屏大图**，并保持后续维护成本最低。

## 1. 资源基线（本项目已验证）

### 1.1 Desktop 侧三文件（单一图标源）

目录：`src/ReadStorm.Desktop/Assets`

- `app.ico`：Windows 可执行文件图标 + Desktop 窗口图标主文件
- `app.png`：Android 图标源文件（构建时自动同步到 Android 资源目录）
- `avalonia-logo.ico`：历史/备用图标（当前链路不作为主入口）

### 1.2 Android 侧两张大图（开屏图）

目录：`src/ReadStorm.Android/Assets`

- `boot.jpg`：竖屏开屏大图（主图）
- `boot_land.jpg`：横屏开屏大图（可选但强烈建议）

---

## 2. 当前项目的处理方式（推荐直接复用）

## 2.1 Windows 图标链路（Desktop）

### A. EXE 文件图标

在 `src/ReadStorm.Desktop/ReadStorm.Desktop.csproj` 中使用：

- `<ApplicationIcon>Assets\app.ico</ApplicationIcon>`

作用：

- 生成的 Windows 可执行文件在资源管理器中显示项目图标
- 任务栏固定/开始菜单等系统入口使用该图标

### B. 窗口图标

在 `src/ReadStorm.Desktop/Views/MainWindow.axaml` 中使用：

- `Icon="/Assets/app.ico"`

作用：

- 程序窗口左上角及任务切换视图中的窗口图标保持一致

---

## 2.2 Android 图标链路（由 Desktop 资源自动驱动）

关键点：**不在 Android 目录手工维护多份图标**，而是构建时从 `Desktop/Assets/app.png` 同步。

在 `src/ReadStorm.Android/ReadStorm.Android.csproj` 中，`SyncAppIcon` 会自动执行：

- `Desktop/Assets/app.png -> Android/Resources/drawable/icon.png`
- `Desktop/Assets/app.png -> Android/Resources/mipmap/ic_launcher.png`
- `Desktop/Assets/app.png -> Android/Resources/mipmap/ic_launcher_round.png`

并通过以下引用生效：

- `AndroidManifest.xml`
  - `android:icon="@mipmap/ic_launcher"`
  - `android:roundIcon="@mipmap/ic_launcher_round"`
- `MainActivity.cs`
  - `Icon = "@mipmap/ic_launcher"`
- `mipmap-anydpi-v26/ic_launcher*.xml`
  - 自适应图标前景指向 `@drawable/icon`

### 这一做法的价值

- 图标真实源只有一份（`Desktop/Assets/app.png`）
- 避免“桌面图标改了，Android 忘改”的跨平台漂移
- CI 和本地构建行为一致，不依赖人工同步

---

## 2.3 Android 开屏大图链路（开屏/登录前视觉）

> 你提到的“开平登录图”在本项目语义上等价于“开屏大图”。

### A. 资源同步

在 `ReadStorm.Android.csproj` 的 `SyncBootImage` 中自动同步：

- `Assets/boot.jpg -> Resources/drawable-nodpi/boot.jpg`
- `Assets/boot.jpg -> Resources/drawable-nodpi/logo.jpg`
- `Assets/boot_land.jpg -> Resources/drawable-nodpi/boot_land.jpg`

说明：

- `drawable-nodpi` 可避免系统按密度二次缩放导致失真
- 竖屏与横屏分图可减少裁切损耗

### B. 启动 Activity 显示策略

`SplashActivity.cs` 负责全屏展示：

- 竖屏优先 `boot`
- 横屏优先 `boot_land`（不存在时回退 `boot`）
- 通过矩阵裁切（`ImageView.ScaleType.Matrix`）实现 **等比填充 + 可控焦点裁切**

这比“简单拉伸”更稳，能兼顾不同长宽比机型。

### C. Android 12+ 系统 Splash 兼容策略

在 `Resources/values-v31/styles.xml`：

- 使用纯色 `windowSplashScreenBackground`
- `windowSplashScreenAnimatedIcon` 指向透明占位 `@drawable/splash_icon_blank`

原因：

- Android 12+ 系统 Splash 会把 icon 限制在小范围，直接塞大图会变小图标
- 先给系统阶段纯色过渡，再由 `SplashActivity` 全屏展示大图，视觉更一致

## 3. 适配到其他项目的最小落地方案（MVP）

1. 在 Desktop 资产目录放置三文件：
   - `app.ico`
   - `app.png`
   - （可选）`avalonia-logo.ico` 作为备用
2. 在 Android 资产目录放置两张开屏图：
   - `boot.jpg`
   - `boot_land.jpg`
3. Desktop 工程：
   - `ApplicationIcon` 指向 `Assets/app.ico`
   - 主窗口 `Icon` 指向 `/Assets/app.ico`
4. Android 工程：
   - 新增 `SyncAppIcon`（图标自动同步）
   - 新增 `SyncBootImage`（开屏图自动同步）
5. Android Manifest / Activity / mipmap XML：
   - 统一引用 `ic_launcher` / `ic_launcher_round` / `@drawable/icon`
6. Android 12+ 样式：
   - 系统 Splash 用纯色 + 透明占位 icon
   - 全屏大图在 `SplashActivity` 显示

---

## 4. 推荐规格与命名约定（跨项目通用）

### 4.1 图标素材

- `app.png`：建议正方形高分辨率（至少 1024x1024）
- `app.ico`：包含多尺寸图层（16/32/48/64/128/256）

### 4.2 开屏大图

- `boot.jpg`（竖屏）：建议 $9:16$
- `boot_land.jpg`（横屏）：建议 $16:9$
- 主题色建议与 `launch_background` 保持一致，减少切换“闪断感”

### 4.3 命名稳定性

尽量固定以下资源名，避免多项目迁移时改一大片引用：

- `app.ico`
- `app.png`
- `boot.jpg`
- `boot_land.jpg`
- `ic_launcher`
- `ic_launcher_round`

---

## 5. 验证清单（发布前必走）

### Desktop

- EXE 文件图标是否为 `app.ico`
- 主窗口标题栏图标是否为 `app.ico`

### Android

- 桌面启动器图标（圆角/非圆角）是否与 `app.png` 一致
- 冷启动时是否先纯色过渡，再进入全屏 `boot.jpg`
- 横屏启动是否优先使用 `boot_land.jpg`
- 不同比例机型是否无明显拉伸变形

---

## 6. 为什么这套方法“极度适合现有框架与跨平台”

- **单一源头**：Desktop 图标资源驱动 Android 图标，减少重复维护
- **构建自动化**：全部由 MSBuild Target 同步，减少手工操作失误
- **平台分层清晰**：系统级 Splash 与应用级全屏图职责分离
- **可迁移**：对业务层零侵入，主要改工程文件与资源链路
- **可回归**：每次构建都会执行同步步骤，行为稳定、易排查

---

## 7. 迁移注意事项（避免踩坑）

- 不要把 Android 图标当手工资源长期维护（容易与 Desktop 脱节）
- 不要在 Android 12+ 把大图直接当系统 Splash icon（会被缩小）
- 不要省略 `boot_land.jpg`（横屏设备会出现过度裁切）
- 不要跳过 `drawable-nodpi` 路径（高密度设备易出现缩放偏差）
