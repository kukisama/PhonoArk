# PhonoArk Pure 项目对比分析报告

> 本文档对仓库根目录下的 **pureopus** 和 **purecodex** 两个纯原生 Android 项目进行全面对比分析。
> 两个项目的共同目标：使用 Kotlin + Jetpack Compose 重写 PhonoArk（Avalonia 跨平台版）的全部 Android 功能。

---

## 一、项目概览

| 维度 | pureopus | purecodex |
|------|----------|-----------|
| **语言** | Kotlin | Kotlin |
| **UI 框架** | Jetpack Compose + Material3 | Jetpack Compose + Material3 |
| **架构模式** | MVVM + Repository + Hilt DI | MVVM + Repository（手动实例化） |
| **数据库** | Room（SQLite） | SharedPreferences（JSON 序列化） |
| **依赖注入** | Hilt（@HiltViewModel / @Singleton） | 无（手动 new） |
| **Kotlin 源文件数** | **42 个** | **5 个** |
| **代码行数（约）** | **~2,900 行** | **~928 行** |
| **单元测试文件** | **4 个（20 个测试用例）** | **0 个** |
| **i18n 资源文件** | 2 套（EN 96 条 + ZH 96 条） | 仅 `app_name`（1 条） |
| **Gradle 配置** | AGP 8.5.2 / Kotlin 1.9.24 / Compose BOM 2024.12.01 | AGP 8.5.2 / Kotlin 1.9.24 / Compose 1.5.15 |
| **文档** | `复查.md`（功能复查清单） | `README.md`（简介） |

## 三、架构与工程质量对比

| 维度 | pureopus | purecodex |
|------|----------|-----------|
| **分层架构** | ✅ 清晰的 Model → Entity → DAO → Repository → ViewModel → UI 六层 | ⚠️ Model → Repository → ViewModel → UI 四层，逻辑集中在少数文件 |
| **依赖注入** | ✅ Hilt 全量注入，@Singleton/@HiltViewModel | ❌ 手动传参，ViewModel 直接实例化 Repository |
| **数据持久化** | ✅ Room + Entity + DAO（关系型，外键约束） | ⚠️ SharedPreferences + JSON 序列化（无事务，无关系约束） |
| **类型安全** | ✅ 独立数据类×10，sealed class 用于反馈类型 | ⚠️ 所有模型放在 1 个文件（Models.kt），enum 简单定义 |
| **关注点分离** | ✅ 每个页面独立 Screen + ViewModel（5+5=10 个文件） | ⚠️ 全部 UI 在 1 个 AppUi.kt（428 行），1 个 AppViewModel（225 行） |
| **国际化方案** | ✅ Android 资源文件 values/strings.xml + values-zh/strings.xml | ❌ 硬编码文案，strings.xml 仅 app_name |
| **单元测试** | ✅ 4 个测试文件，20 个用例（Model + Repository 层） | ❌ 无任何测试 |
| **代码可维护性** | ✅ 文件职责单一，易于定位和修改 | ⚠️ 文件少但单文件过大，修改需关注全局影响 |
| **错误处理** | ✅ try-catch 保护数据加载、类型转换等关键路径 | ⚠️ 最基本的 try-catch，部分路径无保护 |
| **并发控制** | ✅ viewModelScope + Flow + StateFlow | ✅ viewModelScope + StateFlow |

## 五、是否真实可用？

### pureopus ✅ 真实可用

- 所有 30 项功能均已实现并验证（`复查.md` 记录了 4 项关键修复 + 6 项推荐改进均已完成）
- Hilt + Room + Navigation Compose 架构成熟稳定
- 20 个单元测试覆盖核心 Model 与 Repository 层
- 两套完整 i18n 资源（EN / ZH 各 96 条）
- TTS 口音/音量均正确传入 Android TextToSpeech API
- 可直接 `./gradlew assembleDebug` 构建 APK

### purecodex ⚠️ 部分可用

- 基本流程（浏览音标、做测验、查看历史、收藏）可以走通
- 但以下关键功能**未真正生效**：
  - 口音切换：设置存储了但 TTS 始终使用 `Locale.US`
  - 音量控制：设置存储了但 TTS 未接收音量参数
  - 语言切换：设置存储了但 UI 文案硬编码，无法切换
- 缺少收藏分组、测验范围选择、自动跳转、历史展开等进阶功能
- 无测试覆盖，重构风险较高

## 七、结论

| | pureopus | purecodex |
|---|---------|-----------|
| **完成度** | **100%** | **50%** |
| **推荐程度** | ⭐⭐⭐⭐⭐ 可作为正式发布版本 | ⭐⭐⭐ 可作为原型/Demo 参考 |
| **代码质量** | 生产级（分层清晰、DI、测试、i18n） | 原型级（单文件集中、无测试、硬编码） |
| **是否可用** | ✅ 直接可用 | ⚠️ 基本流程可用，进阶功能缺失 |

**pureopus 的完成度显著高于 purecodex**。pureopus 已经达到了与 PhonoArk 原版功能 1:1 对等的水平，具备生产级代码质量；而 purecodex 更像是一个快速原型，核心流程可以走通，但在功能完整度、架构规范、工程质量方面都有较大差距。

如果只保留一个项目继续维护，**建议选择 pureopus**。
